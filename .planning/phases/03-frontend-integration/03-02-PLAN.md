# Plan 03-02: Saxo Portfolio Hook with 60-Second Polling and Signal Enrichment

<objective>
Create the `useSaxoPortfolio` hook that fetches Saxo positions, balance, and performance data from the backend API, enriches mapped positions with TA signals via `getStockSignal`, and silently polls every 60 seconds. This hook is the data layer consumed by the Saxo dashboard UI in plan 03-03.
</objective>

```typescript
import { useState, useEffect, useCallback, useRef } from "react";
import {
  getSaxoPositions,
  getSaxoBalance,
  getSaxoPerformance,
  getStockSignal,
} from "@/lib/api";
import type {
  SaxoPosition,
  SaxoBalance,
  SaxoPerformance,
  ConsolidatedSignalLevel,
} from "@/types";

const POLL_INTERVAL_MS = 60_000;

export interface SaxoPositionEnriched extends SaxoPosition {
  signal?: ConsolidatedSignalLevel;
}

interface SaxoPortfolioState {
  positions: SaxoPositionEnriched[];
  mappedCount: number;
  unmappedCount: number;
  balance: SaxoBalance | null;
  performance: SaxoPerformance | null;
  isLoading: boolean;
  error: string | null;
  pollError: string | null;
  lastRefreshed: Date | null;
}

export function useSaxoPortfolio() {
  const [state, setState] = useState<SaxoPortfolioState>({
    positions: [],
    mappedCount: 0,
    unmappedCount: 0,
    balance: null,
    performance: null,
    isLoading: true,
    error: null,
    pollError: null,
    lastRefreshed: null,
  });

  const isInitialLoad = useRef(true);

  const fetchData = useCallback(async () => {
    const isPoll = !isInitialLoad.current;

    if (!isPoll) {
      setState((prev) => ({ ...prev, isLoading: true, error: null }));
    }

    try {
      const [posResult, balResult, perfResult] = await Promise.allSettled([
        getSaxoPositions(),
        getSaxoBalance(),
        getSaxoPerformance(),
      ]);

      if (posResult.status === "rejected") {
        const errMsg = posResult.reason?.message || "Failed to load Saxo positions";
        if (isPoll) {
          setState((prev) => ({ ...prev, pollError: errMsg }));
          return;
        }
        setState((prev) => ({ ...prev, isLoading: false, error: errMsg }));
        return;
      }

      const posData = posResult.value;
      const balance = balResult.status === "fulfilled" ? balResult.value : null;
      const performance = perfResult.status === "fulfilled" ? perfResult.value : null;

      // Enrich mapped positions with TA signals
      const enriched: SaxoPositionEnriched[] = await Promise.all(
        posData.positions.map(async (pos) => {
          if (!pos.mapped || !pos.yahoo_ticker) {
            return pos;
          }

          try {
            const signalData = await getStockSignal(pos.yahoo_ticker);
            return { ...pos, signal: signalData.consolidated.signal };
          } catch {
            return pos;
          }
        })
      );

      isInitialLoad.current = false;

      setState({
        positions: enriched,
        mappedCount: posData.mapped_count,
        unmappedCount: posData.unmapped_count,
        balance,
        performance,
        isLoading: false,
        error: null,
        pollError: null,
        lastRefreshed: new Date(),
      });
    } catch (err) {
      const errMsg = err instanceof Error ? err.message : "Failed to load Saxo data";
      if (isPoll) {
        setState((prev) => ({ ...prev, pollError: errMsg }));
      } else {
        setState((prev) => ({ ...prev, isLoading: false, error: errMsg }));
      }
    }
  }, []);

  useEffect(() => {
    fetchData();

    const intervalId = setInterval(fetchData, POLL_INTERVAL_MS);

    return () => clearInterval(intervalId);
  }, [fetchData]);

  return {
    ...state,
    refetch: fetchData,
  };
}
```

Key design decisions:
1. **Silent polling**: `isInitialLoad` ref tracks whether this is the first fetch. On poll-triggered fetches, `isLoading` stays `false` — the UI keeps showing existing data.
2. **Poll errors are separate**: `pollError` is distinct from `error`. Initial load failures show `error` (which triggers `<ErrorMessage>` in the UI). Poll failures show `pollError` (which triggers a non-blocking amber banner). On next successful poll, `pollError` clears.
3. **Signal enrichment**: For each position where `mapped === true` and `yahoo_ticker` is present, calls `getStockSignal(yahoo_ticker)` to get the consolidated signal. Uses `Promise.all` with individual try/catch per position — a failed signal fetch for one position does not block others.
4. **`SaxoPositionEnriched` interface**: Extends `SaxoPosition` with an optional `signal` field. This is the type the UI components consume.
5. **Partial success**: If positions succeed but balance or performance fails, the hook still updates with available data (using `Promise.allSettled`).
6. **Cleanup**: The `setInterval` is cleared on unmount, so switching tabs stops polling.
7. **`POLL_INTERVAL_MS = 60_000`**: Matches UI-05 requirement of 60-second polling.

- Polling is silent: `isLoading` is only `true` on initial mount, not on subsequent polls
- `pollError` state is separate from `error` — poll failures do not replace the table with an error
- `lastRefreshed: Date | null` is updated after each successful fetch
- `setInterval` cleanup in `useEffect` return prevents polling after unmount
- `SaxoPositionEnriched` extends `SaxoPosition` with optional `signal?: ConsolidatedSignalLevel`
