# Phase 3 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (shadcn not initialized) |
| Preset | not applicable |
| Component library | none (hand-rolled Tailwind) |
| Icon library | none (text labels and CSS-only indicators used throughout) |
| Font | DM Sans (body + headings), JetBrains Mono (numbers/prices), Instrument Serif (display — not used in this phase) |

> **Design System Split — Implementation Note:**
> A split exists between the dark financial terminal theme defined in `globals.css` (deep navy: `#0a0f1a` / `#0f1629` / `#131b2e`) and the existing `PortfolioDashboard` and `SettingsPage` components which use Tailwind light-mode classes (`bg-white`, `border-gray-200`, `text-gray-900`). The `LoadingSkeleton` component uses dark slate styles (`bg-slate-700/800/900`) that align with `globals.css`, not with the portfolio/settings pages.
>
> **Resolution for Phase 3:** New Saxo components (`SaxoPortfolioDashboard`, `SaxoPositionRow`) MUST match the **existing light-mode pattern** used in `PortfolioDashboard` and `SettingsPage`. Do not introduce dark-mode CSS variables from `globals.css` into new components. This keeps visual consistency within the portfolio and settings sections. The dark terminal theme applies to other areas of the app (charts, stock detail pages). When using `LoadingSkeleton`, use `variant="table-row"` and `variant="card"` — accept the existing dark skeleton as-is since replacing it is out of scope for this phase.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding within badges |
| sm | 8px | Compact element spacing, inner card padding top/bottom |
| md | 16px | Default element spacing, card padding, table cell horizontal padding |
| lg | 24px | Section padding (`p-6`), gap between summary cards and table |
| xl | 32px | Layout gaps between major sections (`space-y-8` in settings) |
| 2xl | 48px | Page vertical padding (`py-8` on settings container, `py-12` on portfolio page) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch targets for tab buttons: minimum 44px height achieved via `pb-3 pt-1` plus text height
- Table row height: `py-3 px-4` per cell (12px/16px) — inherited from existing table pattern
- Summary card grid gap: 16px (`gap-4`) matching existing `PortfolioDashboard` card grid

## Component Specifications

### Tab Navigation

Location: `frontend/src/app/portfolio/page.tsx`, rendered above the active dashboard.

```
[Saxo Positions] [Manual Positions]
─────────────────
```

- Container: `border-b border-gray-200 mb-6`
- Nav: `-mb-px flex gap-6`
- Active tab button: `pb-3 text-sm font-medium border-b-2 border-blue-600 text-blue-600`
- Inactive tab button: `pb-3 text-sm font-medium border-b-2 border-transparent text-gray-500 hover:text-gray-700 hover:border-gray-300 transition-colors`
- Default active tab: "Saxo Positions" (per D-03)
- Tab labels: "Saxo Positions" and "Manual Positions" (exact strings, per D-01)

### Summary Cards (Saxo Tab)

Four cards in a 4-column grid on sm+ screens, 1-column on mobile.

Grid: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-4`

Card shell: `rounded-lg border border-gray-200 bg-white p-4`

Cards (in order):
1. **Account Value** — `total_value` from `/portfolio/balance`, formatted with `formatCurrency` + `balance.currency`
2. **Cash Balance** — `cash_balance` from `/portfolio/balance`
3. **Day P&L** — `change_today` from `/portfolio/performance`, colored green/red based on sign
4. **Total P&L** — `unrealized_positions_value` as a proxy; or compute from positions sum

Card label: `text-sm text-gray-500` (12px → override to `text-xs text-gray-500` to match existing cards)
Card value: `text-2xl font-bold text-gray-900 number-display` (use JetBrains Mono)
P&L sub-line: `text-sm` with color class (`text-green-600` or `text-red-600`) for percentage

### Saxo Positions Table

Column headers (in order): Symbol | Description | Amount | Price | Market Value | P&L | Signal

Column details:
- **Symbol**: `text-left`. Mapped: `<Link href="/stock/{yahoo_ticker}">` with `text-blue-600 hover:underline font-medium`. Unmapped: plain `<span>` with `text-gray-900 font-medium` — not clickable (per D-07).
- **Description**: `text-left text-sm text-gray-500` — `position.description` truncated at 24 chars with CSS `truncate max-w-[140px]`
- **Amount**: `text-right` — quantity held (`position.amount`)
- **Price**: `text-right number-display` — `position.current_price` formatted with `formatCurrency(price, position.currency)`
- **Market Value**: `text-right number-display` — `position.market_value`
- **P&L**: `text-right` — two-line display: `profit_loss_base_currency` (amount) + derived percentage. Color: `text-green-600` or `text-red-600` based on sign.
- **Signal**: `text-center` — `<SignalBadge signal={signal} size="sm" />` for mapped positions. `<span className="text-xs text-gray-400">—</span>` for unmapped (per D-06).

Table shell: `rounded-lg border border-gray-200 bg-white overflow-x-auto`
Table: `w-full min-w-[700px]` (wider than manual table due to extra columns)
Table header: `border-b border-gray-200 bg-gray-50`
Header cells: `px-4 py-3 text-left text-xs font-medium text-gray-500 uppercase`
Body rows: `border-b border-gray-100 last:border-0 hover:bg-gray-50 transition-colors`
Body cells: `px-4 py-3 text-sm`

### Unmapped Position Indicator

Unmapped positions (`mapped: false`) render as a static row with a muted signal cell. No additional visual treatment (badge, icon, strikethrough) is applied to the row itself — the absent signal badge is the sole indicator. This matches D-05 (inline in same table) and D-06 (muted dash).

Tooltip on the dash: `title="TA signals unavailable — instrument not mapped to Yahoo Finance"` as a native HTML tooltip. No custom tooltip component needed.

### "Not Connected" Empty State (Saxo Tab)

Shown when `getSaxoStatus()` returns `connected: false` and the user has not connected.

Container: `rounded-lg border border-gray-200 bg-white p-12 text-center`
Heading: `text-lg font-semibold text-gray-900 mb-2` — "Connect your Saxo account"
Body: `text-sm text-gray-500 mb-6` — see Copywriting Contract below
CTA button: `rounded-lg bg-blue-600 px-6 py-2 text-sm text-white font-medium hover:bg-blue-700` — links to settings page (not a direct OAuth trigger from portfolio page)

### Connection Status Badge (Settings Page)

Displayed inline next to "Saxo Bank" in the Brokerage Connections section.

Connected state: `inline-flex items-center gap-1.5 rounded-full bg-green-100 px-2.5 py-1 text-xs font-medium text-green-700`
- Green dot: `h-1.5 w-1.5 rounded-full bg-green-500`
- Label: "Connected"

Disconnected state: `inline-flex items-center gap-1.5 rounded-full bg-gray-100 px-2.5 py-1 text-xs font-medium text-gray-500`
- Gray dot: `h-1.5 w-1.5 rounded-full bg-gray-400`
- Label: "Not connected"

Circuit breaker tripped: `inline-flex items-center gap-1.5 rounded-full bg-amber-100 px-2.5 py-1 text-xs font-medium text-amber-700`
- Amber dot: `h-1.5 w-1.5 rounded-full bg-amber-500`
- Label: "Re-authentication required"

### Brokerage Connections Section (Settings Page)

New `<section>` appended as third section, before the Save button.

Section shell: `rounded-lg border border-gray-200 bg-white p-6` (identical to existing sections)
Section heading: `text-lg font-semibold text-gray-900 mb-4` — "Brokerage Connections"

Layout:
```
┌─────────────────────────────────────────────────────┐
│ Brokerage Connections                               │
│                                                     │
│ [Saxo Bank logo/text]         [● Connected]         │
│ Connect your Saxo account to view live positions    │
│                                                     │
│ [Connect Saxo Account]  or  [Disconnect]            │
│                                                     │
│ Expires: {expires_at formatted}  (when connected)   │
└─────────────────────────────────────────────────────┘
```

Connect button: `rounded-lg bg-blue-600 px-4 py-2 text-sm text-white font-medium hover:bg-blue-700`
Disconnect button: `text-sm text-red-600 hover:underline` (text-only, no fill — destructive secondary)
Expiry info: `text-xs text-gray-400 mt-2` — shown only when connected

### "Last Updated" Indicator

Rendered below the Saxo positions table when data has been fetched at least once.

`text-xs text-gray-400 mt-2 text-right` — "Updated {N} seconds ago" or "Updated just now"

This derives from `lastRefreshed: Date | null` state in `useSaxoPortfolio`. Updates every second via a secondary `setInterval` in the component (not the hook). Alternatively, a static "Updated at {HH:MM:SS}" is acceptable and simpler — use this if the dynamic "X seconds ago" adds complexity.

### Loading State

Initial load (first mount of `SaxoPortfolioDashboard`):
- Summary cards area: `<LoadingSkeleton variant="card" count={4} />`
- Table area: `<LoadingSkeleton variant="table-row" count={5} />`

Poll-triggered refresh (every 60s after initial load):
- Do NOT reset to loading state. Data remains visible.
- Optionally: show a subtle spinning indicator `animate-spin` in the "Last updated" line only.

### Error State

Use `<ErrorMessage>` component (`frontend/src/components/ui/ErrorMessage.tsx`) for errors on initial load.

For poll-triggered errors (after data was previously loaded):
- Do NOT replace the table with `<ErrorMessage>`.
- Show a non-blocking inline banner above the table:
  `rounded-lg border border-amber-200 bg-amber-50 px-4 py-2 text-sm text-amber-700 mb-3`
  Copy: "Could not refresh Saxo data. Showing last known positions."

## Interaction Contracts

### Tab Switching
- Switching to "Manual Positions" tab unmounts `SaxoPortfolioDashboard`, which clears the 60-second polling interval via `useEffect` cleanup.
- Switching back to "Saxo Positions" tab mounts `SaxoPortfolioDashboard`, triggering a fresh initial fetch.
- No data is cached between tab switches in v1 (acceptable since the 60-second poll interval means data is fresh).

### OAuth Connect Flow
1. User clicks "Connect Saxo Account" in Settings.
2. Button shows loading state: disabled + text "Connecting...".
3. Frontend calls `getSaxoConnectUrl()` → `GET /api/saxo/auth/connect`.
4. On success: `window.location.href = response.auth_url` (full page redirect to Saxo OAuth).
5. After Saxo OAuth: backend redirects to `SAXO_FRONTEND_REDIRECT_URL` (e.g., `/settings`).
6. Settings page mounts, calls `getSaxoStatus()` on load, renders connected badge.

### OAuth Disconnect Flow
1. User clicks "Disconnect".
2. Button shows loading state: disabled + text "Disconnecting...".
3. Frontend calls `disconnectSaxo()` → `DELETE /api/saxo/auth/disconnect`.
4. On success: re-fetch status, update badge to "Not connected", hide Disconnect button.
5. On error: show inline error message, re-enable button.

### 60-Second Silent Poll
- `useSaxoPortfolio` fetches on mount (`isLoading = true`), then polls every 60s (`isLoading` stays false during polls).
- `lastRefreshed` state is updated after each successful fetch.
- Failed poll preserves previous data and shows non-blocking amber banner.
- Poll stops when component unmounts (tab switch or page navigation).

### Mapped vs Unmapped Row Behavior
- Mapped (`mapped: true`): Symbol cell is `<Link href="/stock/{yahoo_ticker}">` — entire symbol is clickable.
- Unmapped (`mapped: false`): Symbol cell is plain text — cursor is default, no click handler.
- No row-level click handler on either type (detail navigation is symbol-cell-only).
